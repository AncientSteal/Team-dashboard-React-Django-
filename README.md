# Team Dashboard — Full-Stack Платформа Управления Командой

Современное веб-приложение для организации командной работы, управления задачами и внутреннего взаимодействия. Проект построен по принципу разделения клиентской и серверной части с использованием актуального технологического стека.

## 🚀 Технологический стек

* **Frontend:** React 19, TypeScript, Tailwind CSS 4, React Router 7.
* **Backend:** Django 6 (Python 3.13), PyJWT (JSON Web Tokens), SQLite / PostgreSQL.
* **Безопасность & Валидация:** Строгая типизация TS, кастомные хуки авторизации и валидации форм, хранение секретов в переменных окружения (`.env`), хэширование паролей.

## ✨ Ключевой функционал

* 🔐 **Полноценная авторизация:** Регистрация, вход и автоматическое восстановление сессии при перезагрузке страницы с помощью JWT-токенов.
* 👤 **Профиль пользователя:** Персональная страница сотрудника с динамической загрузкой аватара и обложки профиля через `FormData` и интеграцией с Pillow.
* 🛠️ **Кастомная архитектура форм:** Облегченный хук `useForm` со строгой типизацией правил валидации на фронтенде для предотвращения отправки некорректных данных.
* 📋 **Система задач (Tasks):** Управление тасками команды (в процессе интеграции интерфейса с бэкенд-моделями).

## 🛠️ Инструкция по локальному запуску

### 1. Клонирование репозитория
```bash
git clone https://github.com
cd team-dashboard
```

### 2. Настройка бэкенда (Django)
Перейдите в папку бэкенда, создайте виртуальное окружение и установите зависимости:
```bash
cd backend
python -m venv venv
source venv/Scripts/activate  # Для Windows: venv\Scripts\activate
pip install -r requirements.txt
```
*Создайте файл `.env` в папке `backend/` и заполните его по шаблону:*
```env
DJANGO_SECRET_KEY="ваш секретный ключ"
DJANGO_DEBUG=True
```
Можно сгенерировать новый ключ комнадой в терминале: 
```
python manage.py shell -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"
```
Выполните миграции и запустите сервер разработки:
```bash
python manage.py makemigrations
python manage.py migrate
python manage.py runserver
```
Бэкенд запустится по адресу: `http://127.0.0.1:8000`

### 3. Настройка фронтенда (React)
Откройте новый терминал в папке проекта:
```bash
cd frontend
npm install
```
*Создайте файл `.env` в папке `frontend/`:*
```env
VITE_API_URL=http://127.0.0.1:8000
```
Запустите клиентское приложение:
```bash
npm run dev
```

Фронтенд запустится по адресу: `http://localhost:5173` (или указанному в терминале Vite).

## 🔒 Безопасность и лучшие практики
* Все чувствительные данные (`SECRET_KEY`, пути базы данных) полностью изолированы из кода и вынесены в переменные окружения, исключая попадание в публичный репозиторий (настроен `.gitignore`).
* Сетевые запросы защищены от зависания интерфейса с помощью `AbortController` с таймаутом в 5 секунд.
* Для работы с динамическими файлами изображений настроен бэкенд-парсер `multipart/form-data` и валидация форматов Pillow.
